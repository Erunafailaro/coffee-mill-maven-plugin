Plugin Documentation

Goals available for this plugin:

GoalReport?Description
coffee-mill:aggregate-javascriptNoAggregate JavaScript sources: Aggregation order can be specified in the Maven Configuration It can also include:
  • The Javascript file from the project ((filename.js)
  • Dependencies files (artifactId.js)
  • Compile files (CoffeeScript) (coffee_filename.js)
If no file are included, the project js files are aggregated in the alphabetical order.
coffee-mill:aggregate-stylesheetsNoAggregate stylesheets sources: Aggregation order can be specified in the Maven Configuration It can also include:
  • The CSS file from the project ((filename.js)
  • Compiled files (LESS) (less_filename.js)
If no file are included, the project CSS files are aggregated in the alphabetical order.
coffee-mill:build-zipNoBuild a zip file including the all assets, generated files and dependencies.
coffee-mill:compile-coffeescriptNoCompiles CoffeeScript files. CoffeeScript files are generally in the src/main/coffee directory. It can be configured using the coffeeScriptDir parameter. If the directory does not exist, the compilation is skipped.
coffee-mill:compile-cssNoCopy CSS to the work directory and check CSS file using CSSLint.
coffee-mill:compile-javascriptNoCopy JavaScript sources to the work directory and check JavaScript sources with
  • Check the code using JSLint
  • Check the code using JSHint
  • Compile dust template (.dust files)
coffee-mill:compile-lessNoCompiles Less files.
coffee-mill:copy-assetsNoCopy src/main/www and src/main/resources to the www directory
coffee-mill:csslint-reportYesBuilds the CSSLint Report.
coffee-mill:helpNoDisplay help information on coffee-mill-maven-plugin.
Call 
  mvn coffee-mill:help -Ddetail=true -Dgoal=<goal-name>
to display parameter details.
coffee-mill:jasmine-it-testNoNo description.
coffee-mill:jasmine-testNoNo description.
coffee-mill:jsdocYesBuilds the JSDoc API. It uses JSDoc3 but requires to have the jsdoc executable in the path.
coffee-mill:jshint-reportYesBuilds the JSHint Report.
coffee-mill:jslint-reportYesBuilds the JSHint Report.
coffee-mill:minify-javascriptNoMinify JavaScript sources. It can use Google Closure or YUI Compressor. The minified file is attached to the project using the min classifier, except if the attachMinifiedJavaScript parameter is set to false
coffee-mill:minify-stylesheetsNoMinify CSS sources. It can use the CSS compressor from YUI. The minified file is attached to the project using the min classifier, except if the attachMinifiedCSS parameter is set to false
coffee-mill:resolve-dependenciesNoCopy `js` project dependencies to the target/libs folder. Copy `css` project dependencies to the target/web folder. The location can be changed using the webDir and libsDir option
coffee-mill:test-compile-coffeescriptNoCompiles CoffeeScript files. CoffeeScript files are generally in the src/test/coffee directory. It can be configured using the coffeeScriptTestDir parameter. If the directory does not exist, the compilation is skipped.
coffee-mill:test-compile-javascriptNoCopy JavaScript sources to the work directory and check JavaScript sources with
  • Check the code using JSLint
  • Check the code using JSHint
TODO Exclude strict mode.
coffee-mill:watchNoThis mojo watches the file change in the source directories and process them automatically. To work correctly, launch mvn clean test first. This will resolve and prepare all required file. Then mvn org.nano.coffee-mill:coffee-mill-maven-plugin:watch will starts the watch mode. This mojo supports reactor mode, i.e. is able to watch several modules and updates files. To enable this mode, launch the watch mode with -Dwatched.project=artifactId of the final project. This will watch all resources of all the modules of the reactor and copy the resulting artifact on the other module in the specified project.

Usage

You should specify the version in your project's plugin configuration:

<project>
  ...
  <build>
    <!-- To define the plugin version in your parent POM -->
    <pluginManagement>
      <plugins>
        <plugin>
          <groupId>org.nano.coffee-mill</groupId>
          <artifactId>coffee-mill-maven-plugin</artifactId>
          <version>1.0.0-SNAPSHOT</version>
        </plugin>
        ...
      </plugins>
    </pluginManagement>
    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>org.nano.coffee-mill</groupId>
        <artifactId>coffee-mill-maven-plugin</artifactId>
        <version>1.0.0-SNAPSHOT</version>
      </plugin>
      ...
    </plugins>
  </build>
  ...
  <!-- To use the report goals in your POM or parent POM -->
  <reporting>
    <plugins>
      <plugin>
        <groupId>org.nano.coffee-mill</groupId>
        <artifactId>coffee-mill-maven-plugin</artifactId>
        <version>1.0.0-SNAPSHOT</version>
      </plugin>
      ...
    </plugins>
  </reporting>
  ...
</project>

For more information, see "Guide to Configuring Plug-ins"
